5.1 Schema

GraphQL 拥有类型系统，每个字段都必须约定一个类型。类型的定义和操作定义是通过 schema 来定义的。GraphQL 的 Schema 语言全称叫 Schema Definition Language。Schema 本身并不代表你数据库中真实的数据结构，它的定义决定了这整个端点能干些什么事情，确定了我们能向端点要什么，操作什么。(代码清单 5-1 定义的 typeDefs 变量就是一个 schema , 至于变量名使用 typeDefs 则是因为和 apollo server 保持一致罢了)。

5.2 类型系统、标量类型、非空类型、参数

  1. 何谓类型系统？
    先看右边的 schema，type 是一个最基本的一个概念，表示一个 GraphQL 对象类型。type 可以包含多个字段(field)。字段类型除了 GraphQL 内置的类型，还可以是 schema 中自定义的 type(便于复用、组合)。例如：上面 schema 中 Query 字符的 movie 类型就是 Movie。
  2. 有哪些内置的标量类型？
    在 GraphQL 中，有如下几种标量类型：Int, Float, String, Boolean, ID，List，Enum。分别表示整型、浮点型、字符串、布尔型，ID类型，列表类型和枚举类型。ID类型代表着一个独一无二的标识，ID类型最终会被转化成String类型，但它必须是独一无二的，例如 mongodb 中的 _id 字段就可以设置为 ID 类型。同时这些标量类型可以理解为 JavaScript 中的原始类型，上面的标量类型同样可以对应 JavaScript 中的 Number, Number, String, Boolean, Symbol 。

    上面分别展示了如何编写列表(List)和枚举(Enum) 类型。
